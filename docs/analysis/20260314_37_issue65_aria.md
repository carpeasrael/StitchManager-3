# Issue #65 Analysis — No ARIA roles or landmarks in HTML

**Date:** 2026-03-14
**Issue:** #65 (high)

## Problem
Main layout containers lack ARIA landmark roles. Screen readers see flat divs with no semantic meaning.

## Already done
All 8 dialogs have role="dialog", aria-modal="true", aria-label. Toast has role="status", aria-live="polite". Burger menu has role="menu"/menuitem. Many buttons have aria-label.

## Gaps to fix
1. **index.html**: Add landmark roles to layout containers
2. **Sidebar.ts**: Add role="navigation" + aria-label to sidebar container
3. **FileList.ts**: Add role="list" to file list container
4. **StatusBar.ts**: Add role="status" to status bar container
5. **FilterChips.ts**: Add aria-pressed to filter buttons

## Approach
- index.html: role="banner" on .app-menu, role="main" on .app-center, role="complementary" on .app-right, role="contentinfo" on .app-status
- Sidebar.ts render(): set role="navigation", aria-label="Ordner" on container
- FileList.ts render(): set role="list" on scroll container, role="listitem" on cards
- StatusBar.ts render(): set role="status" on container
- FilterChips.ts: add aria-pressed attribute to filter buttons
