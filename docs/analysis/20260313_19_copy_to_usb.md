# Analysis: Copy to USB (Issue #19)

## Problem description

Issue #19 requests: "The application should have a function to copy selected stitch files (.pes; .dst; etc) to a connected USB device."

The application already has a USB-Export feature implemented as part of the batch operations system. However, the feature is currently **only accessible when two or more files are selected** — the USB-Export toolbar button is hidden when a single file (or no file) is selected. The issue says "selected stitch files" which naturally includes the single-file case. A user who selects one file and wants to copy it to USB currently has no way to do so from the UI.

Additionally, there is no keyboard shortcut for USB export, and no export action available from the MetadataPanel (the right-side detail view shown when a single file is selected).

## Affected components

### Frontend
- **`src/components/Toolbar.ts`** — `updateButtonStates()` hides all batch buttons (including `toolbar-btn-batch-export`) when `hasMulti` is false (i.e., fewer than 2 files selected). Lines 139-151.
- **`src/main.ts`** — `toolbar:batch-export` event handler reads `selectedFileIds` (multi-select array). When only one file is selected, `selectedFileIds` is empty (the single selection lives in `selectedFileId` instead). Lines 198-218.
- **`src/components/MetadataPanel.ts`** — No USB-export action exists here. This is the primary view for a single selected file.
- **`src/shortcuts.ts`** — No keyboard shortcut for USB export.
- **`src/services/BatchService.ts`** — `exportUsb()` accepts `fileIds: number[]` — already supports a single-element array, no changes needed.

### Backend
- **`src-tauri/src/commands/batch.rs`** — `batch_export_usb` accepts `file_ids: Vec<i64>` — already works with a single file. No backend changes needed.

## Root cause / rationale

The USB-Export feature was built during the batch operations sprint (Sprint 9) and was designed exclusively as a batch operation. The toolbar groups all batch buttons together and hides them all when `hasMulti` is false:

```typescript
const hasMulti = multiCount > 1;
// ...
btn.style.display = hasMulti ? "" : "none";
```

The `toolbar:batch-export` handler in `main.ts` reads from `selectedFileIds`, which is only populated during multi-select (shift-click or ctrl-click). When a user clicks a single file, only `selectedFileId` (singular) is set; `selectedFileIds` remains empty.

The backend `batch_export_usb` command works perfectly fine with a single-element vector — the gap is purely in the frontend visibility and event wiring.

## Proposed approach

### Step 1: Make USB-Export available for single file selection

In `Toolbar.ts`, separate the USB-Export button from the other batch-only buttons. The USB-Export button should be visible when **at least one file is selected** (either via `selectedFileId` for single selection, or `selectedFileIds` for multi-selection).

Change `updateButtonStates()`:
- Keep batch-rename, batch-organize, and batch-ai hidden when `hasMulti` is false.
- Show `toolbar-btn-batch-export` when `hasFile || hasMulti` (i.e., any selection exists).
- Rename the button label contextually: "USB-Export" for single file, "USB-Export (N)" for multi.

### Step 2: Update the event handler to support single-file export

In `main.ts`, modify the `toolbar:batch-export` handler to build the file ID list from both selection sources:
- If `selectedFileIds` has entries (multi-select), use those.
- Otherwise, if `selectedFileId` is set (single select), use `[selectedFileId]`.
- If neither is set, return early (no-op).

### Step 3: Add USB-Export button to MetadataPanel

Add a secondary "USB-Export" action button in the MetadataPanel's action bar (near the existing "KI analysieren" button) so users can export from the detail view. This button emits `toolbar:batch-export`, reusing the same handler.

### Step 4: Add keyboard shortcut

Add `Cmd/Ctrl+Shift+U` as a keyboard shortcut for USB export in `shortcuts.ts`. The shortcut emits `toolbar:batch-export`, which the updated handler will resolve to whatever is currently selected.

### Step 5: Toast feedback for single-file export

The existing `BatchDialog` progress modal works for multi-file exports but may feel heavy for a single file copy. For single-file exports (1 file), use a simple Toast notification ("Datei exportiert") instead of the full batch dialog. For 2+ files, keep the existing BatchDialog.

### Summary of changes

| File | Change |
|------|--------|
| `src/components/Toolbar.ts` | Show USB-Export button for single selection too |
| `src/main.ts` | Build file ID list from both `selectedFileId` and `selectedFileIds` |
| `src/components/MetadataPanel.ts` | Add USB-Export action button |
| `src/shortcuts.ts` | Add Cmd/Ctrl+Shift+U shortcut |

No backend changes are required — the Rust `batch_export_usb` command already handles single-file vectors correctly.
