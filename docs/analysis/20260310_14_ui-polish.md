# Analysis: Issue #14 — UI polish: folder border shift, missing favicon, dark theme body background, version mismatch

## Problem description

Four minor UI/config issues that degrade visual polish and developer experience:

1. **Folder item selection causes layout shift** — `.folder-item.selected` adds a 1px border that the base state lacks, causing content to shrink by 1px on each side when selected (visible text jitter).
2. **Missing favicon** — `index.html` references `/vite.svg` (Vite template leftover), causing a 404 on every page load.
3. **White flash in dark mode** — Neither `html` nor `body` has `background` or `color` set. Only `.app-layout` uses `var(--color-bg)`. The body remains white behind the layout, visible during elastic scroll or reflows.
4. **Version mismatch** — `package.json` says `0.1.0`, `src-tauri/tauri.conf.json` says `2.0.0` (copy-pasted from Tauri framework version).

## Affected components

- `src/styles/components.css` — `.folder-item` rules (lines 48-66)
- `index.html` — favicon `<link>` (line 5)
- `src/styles.css` — missing `html, body` rule
- `src-tauri/tauri.conf.json` — version field (line 4)

## Root cause / rationale

1. The `.folder-item` base rule has no border. When `.selected` adds `border: 1px solid`, `box-sizing: border-box` shrinks the content area.
2. The favicon link is a leftover from the Vite project template and was never updated.
3. The CSS architecture sets background only on `.app-layout`, not on the document root elements.
4. The Tauri config version was left at the template default `2.0.0`.

## Proposed approach

### Fix 1: Folder border shift
Add `border: 1px solid transparent` to the base `.folder-item` rule in `src/styles/components.css`. This reserves the border space so the selected state only changes the color, not the geometry.

### Fix 2: Missing favicon
Remove the `<link rel="icon" ...>` line from `index.html`. A Tauri desktop app doesn't need a favicon (the window icon is set via Tauri config).

### Fix 3: Dark theme body background
Add a rule in `src/styles.css` after the imports:
```css
html, body {
  background: var(--color-bg);
  color: var(--color-text);
}
```

### Fix 4: Version mismatch
Change `src-tauri/tauri.conf.json` version from `"2.0.0"` to `"0.1.0"` to match `package.json`.
