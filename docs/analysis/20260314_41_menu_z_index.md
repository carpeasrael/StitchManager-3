# Analysis: Ensure all menus are in front when open

**Date:** 2026-03-14

---

## Problem description

The burger menu (hamburger menu in the toolbar) opens behind the sidebar, center, and right panels. This happens because all layout panels (`.app-menu`, `.app-sidebar`, `.app-center`, `.app-right`, `.app-splitter-*`, `.app-status`) have `position: relative; z-index: 1`, which creates stacking contexts. The burger menu is appended as a child of the toolbar element (inside `.app-menu`), so despite having `position: fixed; z-index: 90`, it is trapped inside `.app-menu`'s stacking context (effective root z-index: 1). The sidebar/center/right panels paint later in DOM order and cover the burger menu.

The SearchBar's advanced filter panel does NOT have this problem because it appends to `document.body`, escaping the stacking context.

## Affected components

- `src/styles/layout.css` — z-index: 1 on all layout panels creates unwanted stacking contexts
- `src/components/Toolbar.ts:261` — appends burger menu to `this.el` instead of `document.body`

## Root cause / rationale

The `z-index: 1` on all layout panels was added to ensure they sit above the `.app-layout::before` pseudo-element (background image blur layer). However, this creates stacking contexts that trap fixed-positioned children. The burger menu's `z-index: 90` only applies within `.app-menu`'s stacking context, making it ineffective against sibling panels at the same root z-index.

## Proposed approach

1. **Fix `.app-layout::before`**: Set `z-index: -1` on the pseudo-element to push it behind all content without requiring panels to have z-index.
2. **Remove `z-index: 1` from layout panels**: Remove the z-index from `.app-menu`, `.app-sidebar`, `.app-center`, `.app-right`, `.app-splitter-*`, `.app-status` so they don't create stacking contexts. Keep `position: relative`.
3. This allows the burger menu's `position: fixed; z-index: 90` to participate in the root stacking context and properly overlay all content.
