# Analysis: Mass Import Function (Issue #21)

## Problem Description

The application needs a dedicated **mass import** function that allows users to select a folder, automatically adds it to the application, scans it for embroidery files, imports all found files, and displays:

1. **Progress percentage** during the scan and import process
2. **Estimated/elapsed runtime** so the user knows how long the operation takes

Currently, the workflow requires two separate manual steps (add folder, then scan folder), provides no percentage-based progress feedback, and does not calculate or display runtime information.

## Root Cause / Rationale

### Current Workflow (Two-Step, No Progress)

1. **Step 1 -- Add folder:** User clicks "Ordner hinzufuegen" in toolbar, selects a directory via OS file dialog. This creates a folder record in the DB but does NOT scan or import any files.
2. **Step 2 -- Scan folder:** User must then select the folder in the sidebar, click "Ordner scannen". This calls `scan_directory` (walks the filesystem, emits `scan:file-found` events) and then `import_files` (parses each file, inserts into DB, generates thumbnails).

### What Exists

- **`scan_directory` (backend):** Walks a directory tree, finds embroidery files (PES/DST/JEF/VP3). Emits `scan:progress` every 50 files with `{ current, file }` -- but `current` is the count of ALL files scanned (not just embroidery files) and there is NO total count (so no percentage is possible). Emits `scan:file-found` for each embroidery file. Emits `scan:complete` with the count at the end.
- **`import_files` (backend):** Takes a list of file paths + folder ID, parses each file, inserts into DB, generates thumbnails. Runs synchronously with NO progress events at all -- the frontend blocks until the entire import is done.
- **`BatchDialog` (frontend):** A reusable progress dialog with a progress bar, percentage, step counter ("X von Y Dateien"), and a log of per-file results. It listens for `batch:progress` events with `{ current, total, filename, status }`. This is exactly what we need for import progress display.
- **`BatchProgressPayload` (backend batch.rs):** The batch system already emits per-file progress events with `{ current, total, filename, status }`. The import system needs to do the same.

### Gaps

1. **No combined action:** User must manually add a folder, then manually scan it. The issue requests a single "mass import" action.
2. **No import progress events:** `import_files` does not emit any progress events. For large directories (hundreds or thousands of files), the UI appears frozen.
3. **No percentage:** `scan_directory` emits `scan:progress` with a `current` count but no `total` (unknown until scan completes). The import phase has zero progress reporting.
4. **No runtime calculation:** Neither scan nor import tracks or reports elapsed time or estimated remaining time.

## Proposed Approach

### Step 1: New Backend Command `mass_import`

Create a new Tauri command `mass_import` in `scanner.rs` that combines the folder-add + scan + import into a single operation with granular progress events.

**Signature:**
```rust
#[tauri::command]
pub async fn mass_import(
    db: State<'_, DbState>,
    thumb_state: State<'_, ThumbnailState>,
    path: String,
    app_handle: AppHandle,
) -> Result<MassImportResult, AppError>
```

**Behavior:**
1. Validate the directory path exists
2. Create folder record in DB (or find existing one for that path)
3. **Phase 1 -- Discovery:** Walk the directory tree, collect all embroidery file paths. Emit `import:discovery` events with `{ scannedFiles, foundFiles }` every 50 files. This phase is fast.
4. **Phase 2 -- Import:** For each found embroidery file, parse it, insert into DB, generate thumbnail. Emit `import:progress` event after each file with:
   ```rust
   ImportProgressPayload {
       current: u32,        // files processed so far
       total: u32,          // total embroidery files found
       filename: String,    // current file being processed
       status: String,      // "ok" or "error:..."
       elapsed_ms: u64,     // milliseconds since import started
       estimated_remaining_ms: u64, // extrapolated from avg time per file
   }
   ```
5. Emit `import:complete` with final summary
6. Return `MassImportResult { folder_id, imported_count, skipped_count, error_count, elapsed_ms }`

**Key design decisions:**
- Split into discovery + import phases so we know the total count before starting the import (enabling percentage calculation)
- Emit per-file progress (not every-50) since import is slower than scan and each file matters
- Calculate elapsed and estimated remaining time server-side for accuracy
- Use `INSERT OR IGNORE` to skip already-imported files (same as current `import_files`)

### Step 2: Register Command

In `src-tauri/src/lib.rs`, add `mass_import` to the `.invoke_handler(tauri::generate_handler![...])` list.

### Step 3: Frontend Service Method

Add to `ScannerService.ts`:
```typescript
export interface MassImportResult {
  folderId: number;
  importedCount: number;
  skippedCount: number;
  errorCount: number;
  elapsedMs: number;
}

export async function massImport(path: string): Promise<MassImportResult> {
  return invoke<MassImportResult>("mass_import", { path });
}
```

### Step 4: Extend BatchDialog for Import Progress

Enhance `BatchDialog` to support runtime display:
- Add an elapsed time display (e.g., "Laufzeit: 0:23")
- Add an estimated remaining time display (e.g., "Verbleibend: ~1:45")
- These are updated via the `import:progress` event payload which includes `elapsed_ms` and `estimated_remaining_ms`
- The existing progress bar, percentage, and per-file log already work and can be reused as-is

### Step 5: Add "Massenimport" Toolbar Button

In `Toolbar.ts`:
- Add a new button "Massenimport" (icon: could reuse folder + scan combo icon) that is always enabled (no folder selection required)
- On click: open OS directory picker, then call `ScannerService.massImport(path)`
- Before calling, open `BatchDialog` in "discovery" mode (indeterminate progress)
- Switch to determinate progress once `import:progress` events arrive

### Step 6: Wire Events in `main.ts`

Add Tauri event listeners for:
- `import:discovery` -- forward to EventBus
- `import:progress` -- forward to EventBus, update BatchDialog
- `import:complete` -- forward to EventBus, reload files, refresh sidebar counts

Add EventBus handler for `toolbar:mass-import` that:
1. Opens the directory picker dialog
2. Opens BatchDialog with title "Massenimport"
3. Calls `ScannerService.massImport(path)`
4. On completion: reloads folders, selects the new folder, reloads files, shows toast with summary including elapsed time

### Step 7: Update Types

Add to `src/types/index.ts`:
```typescript
export interface ImportProgress {
  current: number;
  total: number;
  filename: string;
  status: string;
  elapsedMs: number;
  estimatedRemainingMs: number;
}

export interface MassImportResult {
  folderId: number;
  importedCount: number;
  skippedCount: number;
  errorCount: number;
  elapsedMs: number;
}
```

### Implementation Order

1. Backend: Add `mass_import` command with progress events (scanner.rs)
2. Backend: Register command (lib.rs)
3. Frontend: Add types (types/index.ts)
4. Frontend: Add service method (ScannerService.ts)
5. Frontend: Extend BatchDialog with time display
6. Frontend: Add toolbar button (Toolbar.ts)
7. Frontend: Wire events and handler (main.ts)
8. Test with real embroidery files directory
