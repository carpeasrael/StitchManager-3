# Analysis — Issue #43: Burger Menu

Date: 2026-03-13

## Problem Description

The app header currently shows "StichMan" text plus a theme toggle button in a 28px `.app-menu` bar. All action buttons (12 total) are displayed in the `.app-toolbar` row (48px) alongside the search bar and format filter chips. This creates a crowded toolbar that doesn't scale well as features are added.

Issue #43 requests: "Passe die Einträge der Funktionen in der Kopfzeile an und packe die in ein Burger Menü."

The toolbar actions should be moved into a hamburger menu in the `.app-menu` bar, freeing the toolbar row for search and filters only.

## Affected Components

| File | Change |
|------|--------|
| `src/components/Toolbar.ts` | Refactor into a burger menu dropdown instead of inline buttons |
| `src/main.ts` | Update `initComponents()` — mount Toolbar in `.app-menu` instead of `.app-toolbar` |
| `src/styles/layout.css` | Remove the toolbar row from the grid (merge menu+toolbar or reduce toolbar height) |
| `src/styles/components.css` | Add burger menu styles (hamburger icon, dropdown panel, menu items) |
| `index.html` | Remove `.app-toolbar` div (or repurpose for search/filters only) |
| `src/components/SearchBar.ts` | No code changes, but will now live in `.app-menu` or a combined bar |
| `src/components/FilterChips.ts` | Same — repositioned but not changed |

## Root Cause / Rationale

The current flat toolbar with 12 action buttons is space-inefficient. A burger menu is a standard UI pattern that:
- Declutters the header by hiding infrequent actions behind a single toggle
- Groups related actions logically (File, Batch, AI, System)
- Scales better as new features are added

## Proposed Approach

### Step 1: Merge menu and toolbar into a single header bar

- Collapse the two-row layout (`28px menu + 48px toolbar`) into a single row (`40px`)
- Grid changes: `grid-template-rows: 40px 1fr 22px` (3 rows instead of 4)
- Grid areas: remove "toolbar" row, expand "menu" to hold search + filters + burger

### Step 2: Restructure `.app-menu` layout

New internal layout of `.app-menu`:
```
[☰ Burger] [StichMan] [--- SearchBar ---] [FilterChips] [◐ Theme]
```

- Burger button: leftmost, opens dropdown menu
- App title: "StichMan"
- SearchBar component: flex-grow in the center
- FilterChips: after search
- Theme toggle: rightmost (existing)

### Step 3: Refactor `Toolbar.ts` into a burger menu

Convert the `Toolbar` component to render:
1. A hamburger button (`☰`) that toggles a dropdown overlay
2. A dropdown panel with grouped menu items:

| Group | Items |
|-------|-------|
| **Ordner** | Ordner hinzufügen, Ordner scannen, Massenimport |
| **Datei** | Speichern, Im Ordner anzeigen, PDF Export |
| **KI** | KI Analyse, Batch KI |
| **Batch** | Batch Umbenennen, Batch Organisieren, USB-Export |
| **System** | Einstellungen |

- Each menu item shows icon + label + keyboard shortcut (where applicable)
- Disabled/hidden state logic remains the same as current `updateButtonStates()`
- Click outside or Escape closes the menu
- Clicking a menu item closes the menu and fires the same EventBus event

### Step 4: CSS for burger menu

- `.burger-btn`: hamburger icon button, same style as theme toggle
- `.burger-menu`: absolute-positioned dropdown, anchored below the burger button
- `.burger-menu-group`: section with header label + divider
- `.burger-menu-item`: flex row (icon + label + shortcut hint), hover highlight
- `.burger-menu-item:disabled`: grayed out
- Close on click-outside via document-level click handler

### Step 5: Update `index.html`

- Remove the `.app-toolbar` div
- Move search/filter containers into `.app-menu` (done dynamically in `initComponents()`)

### Step 6: Update keyboard shortcuts display

- Show shortcut hints in menu items (e.g., "Speichern  Ctrl+S")
