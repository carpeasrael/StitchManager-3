# Analysis — Issue #44: Cleansing

Date: 2026-03-13

## Problem Description

The application contains hardcoded example/default values that pre-populate the database and UI when the app is first compiled and run. Issue #44 requests removing all example content so the application starts completely empty.

## Affected Components

| File | Change |
|------|--------|
| `src-tauri/src/db/migrations.rs` | Remove 10 default settings INSERT statements (lines 209-218) |
| `src/components/SettingsDialog.ts` | Remove fallback values that populate empty fields with example paths |
| `src-tauri/src/commands/files.rs` | Remove `~/Stickdateien` fallback (line 654) |

## Root Cause / Rationale

Default settings like `library_root = '~/Stickdateien'` and `ai_url = 'http://localhost:11434'` are convenience values from development. In a release build, the app should start with no pre-configured paths or AI settings, forcing the user to configure everything through the Settings dialog.

## Proposed Approach

### Step 1: Remove default settings from migrations

Remove the 10 `INSERT OR IGNORE INTO settings` lines (209-218) from `apply_v1()` in `migrations.rs`. Keep only the essential non-example defaults:
- `theme_mode = 'hell'` — the app needs a theme to render correctly
- `font_size = 'medium'` — if this setting exists (check if it's used)

All other settings (`library_root`, `metadata_root`, `ai_*`, `rename_pattern`, `organize_pattern`) should not be pre-populated.

### Step 2: Update SettingsDialog fallbacks

In `SettingsDialog.ts`, change fallback `||` values to empty strings so empty fields show as empty rather than populated with example values. Keep `placeholder` attributes as hints (these are non-functional UI guidance, not example data):

- `libraryInput.value = settings.library_root || ""` (was `"~/Stickdateien"`)
- `metaInput.value = settings.metadata_root || ""` (was `"~/Stickdateien/.stichman"`)
- `urlInput.value = settings.ai_url || ""` (was `"http://localhost:11434"`)
- `modelInput.value = settings.ai_model || ""` (was `"llama3.2-vision"`)
- `tempSlider.value = settings.ai_temperature || "0.3"` — keep as sensible default
- `timeoutInput.value = settings.ai_timeout_ms || "30000"` — keep as sensible default
- `renameInput.value = settings.rename_pattern || ""` (was `"{name}_{theme}"`)
- `organizeInput.value = settings.organize_pattern || ""` (was `"{theme}/{name}"`)

Placeholders (the grayed-out hint text) are acceptable to keep — they show format examples without populating the field.

### Step 3: Remove backend fallback

In `files.rs` line 654, change `.unwrap_or_else(|_| "~/Stickdateien".to_string())` to return an error or empty string when `library_root` is not set. The app should handle a missing `library_root` gracefully (e.g., skip PDF report path resolution, or prompt user to set it).

### Step 4: Ensure app starts empty

Verify that with no default settings, the app:
- Opens with an empty sidebar (no folders)
- Shows an empty file list
- Settings dialog shows empty fields with placeholder hints
- No errors or panics from missing settings
